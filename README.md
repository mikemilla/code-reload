# Bunbu

On-device live code editor and AI agent for React Native. Edit TSX on your phone, save, and see the preview update — no Metro connection required after install.

## Repo layout

```
bunbu/                 ← this package (ios/, android/, src/)
example/               ← demo app (start here)
  src/App.tsx          ← edit this
  appSources.ts        ← auto-generated snapshot (gitignored)
```

## Prerequisites

- [React Native dev environment](https://reactnative.dev/docs/set-up-your-environment) (Xcode for iOS, Android Studio for Android)
- Node.js ≥ 22.11
- CocoaPods (for iOS)

## First-time setup

From the repo root:

```sh
cd example
npm install
cd ios && pod install && cd ..
```

## Run the example app

Use **two terminals** from `example/`, or run Metro in the background first.

**Terminal 1 — Metro**

```sh
cd example
npm start
```

`prestart` runs `bunbu-snapshot` automatically, which copies `src/**/*.tsx` into a bundled snapshot for the on-device editor.

**Terminal 2 — iOS (physical device or simulator)**

```sh
cd example
npm run ios -- --scheme BunbuFreshRN
```

The example Xcode scheme is `BunbuFreshRN` (not the default `BunbuExample`).

**Android**

```sh
cd example
npm run android
```

## Edit the app

1. Change **`example/src/App.tsx`** in your editor.
2. Reload the preview:
   - **With Metro:** save and use Fast Refresh, or press `R` in the dev menu.
   - **On device (Bunbu Files tab):** open `App.tsx`, edit, tap **Save** — works offline after the app is installed.

Do not edit `example/appSources.ts`; it is generated by `npm run snapshot` and gitignored.

## Bunbu on device

In `__DEV__`, the app mounts Bunbu: tap the floating button to open Files, Chat, History, and Git.

If `App.tsx` is missing from Files after a rebuild, you may see **“Bundled code changed”** — choose **Replace** to seed from the latest snapshot, or delete and reinstall the app.

## Integrate into your app

```js
import { registerBunbuApp, AI_MODELS } from 'bunbu';
import App from './src/App';
import { appSources } from './appSources';

registerBunbuApp('MyApp', App, {
  sources: appSources,
  entry: 'App',
  apiKey: 'your-key-here',
  model: AI_MODELS.claude4Sonnet,
});
```

Generate the snapshot from your source tree:

```sh
node ../scripts/bunbu-snapshot.js --src src --out appSources.ts
```

Or use the `bunbu-snapshot` bin from this package.

## Package scripts (repo root)

```sh
npm run typescript   # typecheck bunbu package
npm run snapshot     # snapshot with default paths (src → src/appSources.ts)
```

## Troubleshooting

| Issue | Fix |
|-------|-----|
| `Could not load the shared scheme for BunbuExample` | Use `--scheme BunbuFreshRN` |
| Pods path errors after moving the repo | `cd example/ios && rm -rf Pods build && pod install` |
| Metro port in use | Stop the other process on `:8081` or run `npx react-native start --port 8082` |
| Stale on-device files | Reinstall the app or tap **Replace** when prompted |

More: [React Native troubleshooting](https://reactnative.dev/docs/troubleshooting)
